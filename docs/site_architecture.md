# Site Architecture

## Public Routes

- `/` — Landing page
- `/presentation/` — Full-screen presentation mode
- `/case-studies/` — Three system case studies
- `/evidence/` — Sanitized technical evidence layer
- `/reflection/` — Cross-course synthesis and future direction

## Reader Journey

identity → thesis → systems → evidence → synthesis → future direction

## Case Study Pattern

Each system should follow:

1. Problem
2. Why it mattered
3. Data/system challenge
4. Method
5. Workflow/control design
6. Result/impact
7. Skills demonstrated
8. Evidence artifacts
